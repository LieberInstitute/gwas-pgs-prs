## Unexpected Issues And Corrections

Initial `--mind 0.02` before variant filtering removed all samples. Cause: the MBv genotype VCF is a union across imputation batches/arrays, so many variants are structurally absent in some samples. Correction: apply variant missingness first, then sample missingness on the shared variant set.

PLINK2 BCF export changed contigs from `chr1` style to `1` style. This caused `bcftools +score` to match zero GWAS-VCF PGS markers. Correction: use PLINK only to determine QC-passing variant IDs, then rebuild a chr-prefixed BCF from the normalized VCF for `bcftools +score`.

PRSice required a phenotype file for BGEN target input even with `--no-regress`. Correction: provide a dummy phenotype file and keep `--no-regress`.

PRSice interpreted BGEN sample IDs as `BrXXXX_BrXXXX`. Correction: use `--ignore-fid` and a dummy phenotype with the same duplicate-form IID, then normalize duplicate IDs back to `BrXXXX` in merged outputs.

PRSice failed on a duplicated base SNP ID in BPD. Correction: deduplicate PRSice base files by first `CHROM:POS:REF:ALT` occurrence before running PRSice.

The earlier DS warning at `chr1:3393935` did not produce any DS out-of-range sites after the normalized autosomal SNP filter. The final DS out-of-range count was 0.

## Production Notes

Production workflows should include hard gates for:
- sample ID equality between phenotype/demographics and genotype files
- contig naming compatibility before scoring
- variant ID uniqueness in target and base score files
- exact allele orientation after any PLINK export
- DS field presence and DS range
- variant missingness before sample missingness for union-imputed files
- explicit exclusion and reporting of ambiguous A/T and C/G SNPs in PRSice
- external LD panel sample ancestry and variant ID convention
- row-count checks after every merged score table
- no zero-marker scoring runs

Do not rely on a PLINK-exported VCF/BCF as the final scoring BCF unless contig names and REF/ALT matching are revalidated against the GWAS-VCF/PGS files.
